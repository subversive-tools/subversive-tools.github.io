Plugins

❖ redmine_subskills

Skill Management & Role Matrix

Transform Redmine into a comprehensive competence management tool. Track employee skills, define required roles, and find the perfect match for project requirements.

For: HR, project managers optimizing team resource allocation
Status: ✅ Stable (v1.0.0, Redmine 4.0+)

  • Central skill catalog management
  • Personal skill profiles for users
  • Automatic role-fit calculation
  • Visual skill matrix for teams
  • Project skill requirements & "Best Match" finder

▽ redmine_submenus

Dropdown Navigation & Portfolio Kanban

Transform your Redmine navigation with intelligent dropdown menus and powerful portfolio management. What starts as enhanced navigation evolves into a comprehensive portfolio dashboard with kanban views.

For: Teams managing multiple subprojects who need clear portfolio oversight
Status: ✅ Stable (v0.5.4, Redmine 5.0–6.0)

  • Project dropdowns appear next to project titles
  • Wiki dropdowns for quick access to child pages
  • Kanban view transforms project lists into interactive dashboards
  • Macro support: {{subprojects(view=kanban)}}

☰ redmine_subnavigation

Collapsible Sidebar Navigation

A powerful, collapsible sidebar navigation tree for Redmine. Navigate smoothly through projects, wiki pages, and headings with a clean, hierarchical view.

For: Documentation-heavy projects needing TOC-like navigation
Status: ✅ Stable (v0.5.4, Redmine 5.0–6.0)

  • Project hierarchy with expand/collapse
  • Wiki tree structure with automatic TOC
  • Smart state persistence between page loads
  • Recursive expansion (Alt/Option + click)
  • Hide breadcrumbs option for cleaner interface

⬡ redmine_subsimplify

Simplified UI for Specific Roles

Provides a distraction-free Redmine interface for specific user roles or groups. Hides unnecessary menus, sidebars, and explicitly restricts access to modules beyond "Issues" and "Wiki".

For: Teams with external collaborators or stakeholders who need a focused, clean view
Status: ✅ Stable (v0.2.6, Redmine 5.0–6.0)

  • Role-based UI simplification
  • Hide sidebar, footer, top menu elements
  • Strict module whitelisting
  • Smart redirect from Overview to first allowed module
  • Profile privacy controls

GitHub  |  Documentation

redmine_subwikifiles

Bidirectional Wiki-to-Markdown Sync

Enable bidirectional synchronization between Redmine wiki pages and local Markdown files. Edit documentation in your favorite external editor; the plugin handles structure, orphan detection, and automatic Git versioning.

For: Teams wanting to edit documentation outside Redmine (VS Code, Obsidian) while maintaining versioning
Status: ⚠️ Beta (v0.4.0, Redmine 5.0–6.0)

  • Wiki pages saved as .md files on filesystem
  • Bidirectional sync (write in Redmine or filesystem)
  • Hierarchical structure via folders
  • Git integration with automatic commits
  • Attachment sync to _attachments folder
  • Orphan file & folder detection

Installation

Generic installation process (applies to all plugins):

cd /path/to/redmine/plugins
git clone https://github.com/modoq/<plugin-name>.git
bundle install
bundle exec rake redmine:plugins:migrate RAILS_ENV=production
touch tmp/restart.txt  # or restart via Docker

Plugin-specific documentation is available in each repository's README.
